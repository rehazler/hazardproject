How It Works

The wiki is powered by a Notion database. Every row in the database becomes one page on the wiki. There are two parts to each entry:

  • Properties — the structured fields on the right side panel (name, category, stats, etc.). These appear in the profile sidebar on the wiki.
  • Page body — the open writing area inside the entry page. You can add anything here: backstory, images, audio clips, tables, and more. This becomes the main content area of the wiki page.
Tip: Think of properties as the character sheet, and the page body as a free-form notebook page attached to it.

Creating a New Entry

1
Open the Voidverse Wiki database in Notion.
2
Click + New at the bottom of the table, or the blue New button in the top right.
3
Fill in the Name field — this is the title that appears on the wiki.
4
Set Campaign and Category so the wiki knows where to list this entry.
5
Optionally add a Profile Image URL and any stat fields.
6
Click the entry to open the page, then write the main content in the body area below the properties.
Tip: Changes appear on the wiki immediately — just refresh the wiki page after saving in Notion.

Database Properties

These fields are visible when you click on an entry's row. They appear in the profile sidebar on the right side of the wiki page.

Property Type What it does
Name Title The entry's name as it appears on the wiki. Required.
Campaign Select Which campaign this entry belongs to. Used to group entries on the wiki's campaign list.
Category Select The type of entry — e.g. NPCs, Locations, Items, Factions. Used to group entries within a campaign.
Profile Image URL A direct link to the entry's profile picture. Paste a full image URL (must start with https://). See the Images section for tips.
Any other property Any Any additional property you add automatically appears in the profile sidebar. Use ALL CAPS names for compact stat attribute boxes, mixed case for body rows. Prefix with a number to control sort order. See Property Naming & Display for the full guide.

Adding Custom Stat Fields

To add a stat like "Race" or "Affiliation":

  1. Click the + button at the end of the property list in the database.
  2. Choose a type — Text for freeform values, Select for a fixed list of options, Checkbox for yes/no.
  3. Name it and it will appear in every entry's sidebar automatically.
Tip: Properties you leave blank for a specific entry won't show up in that entry's sidebar — only filled-in values are displayed.

Property Naming & Display

Any property you add beyond the four reserved ones (Name, Campaign, Category, Profile Image) shows up in the entry's sidebar. The wiki uses the property name to decide how and where to display it.

Two display sections

The sidebar has two distinct areas:

  • Stat attribute boxes — a compact row of labelled value boxes at the top, like a character sheet. Best for short, important values: race, class, HP, AC.
  • Body rows — a simple key: value table below the stat boxes. Best for longer or less critical details: alignment, hometown, pronouns, status.

This same split applies in both the normal sidebar view and the Stat Block toggle view on each entry page.

Rule 1 — ALL CAPS name → stat attribute box

If a property name contains no lowercase letters, it is treated as a stat attribute and placed in the compact top row.

Tip: Numbers, slashes, and spaces are allowed in ALL CAPS names — HP/MAX, CR, and SAVE DC all count as stat attributes.

Rule 2 — Numeric prefix → sort order (stripped on display)

The Notion API does not return properties in column order, so reordering columns in Notion has no effect on the wiki. Instead, prefix the property name with a number followed by a space to set its position explicitly. The number is stripped from the display label.

Tip: Properties without a number prefix always appear after all numbered ones in their section, sorted alphabetically.

Page Body & Blocks

When you open an entry (click to expand it into a full page), the area below the properties panel is the page body. You can add any content here — it becomes the main section of the wiki page.

Every piece of content you add is called a block. To add a block, click into the page and start typing, or press / to open the block menu and search for a type.

Text & Structure

BlockHow to add itWhat it does
Heading 1 / 2 / 3 Type #, ##, or ### then space — or use /heading Section headers. Makes the page easy to scan.
Text (Paragraph) Just start typing Regular body text. Supports bold, italic, underline, strikethrough, and inline links.
Bulleted List Type - then space — or /bullet An unordered list of items.
Numbered List Type 1. then space — or /numbered A numbered list of steps or items.
Toggle Type > then space — or /toggle A collapsible section. Readers click to expand. Great for backstory, spoilers, or long lore sections.
Toggle Heading Add a heading, then click the ⋮⋮ handle → Turn into → Toggle Heading A section header that can be collapsed. Keeps long pages tidy.
Quote Type " then space — or /quote A styled indented quote. Good for in-character dialogue or notable lines.
Callout /callout A highlighted box with an optional emoji icon. Good for DM notes, warnings, or important lore flags.
Divider Type --- — or /divider A horizontal line to separate sections visually.
Table /table A data table. Good for spell lists, item stats, relationship grids, etc.
Columns /columns Side-by-side layout. Drag blocks into columns to arrange them.

Media

BlockHow to add itWhat it does
Image /image Displays an image. You can upload a file or paste a URL. Add a caption below for artist credit or context. See the Images section for tips on URLs.
Audio /audio Displays a small themed play button on the wiki. Paste a direct link to a hosted .mp3, .ogg, or .wav file. Click the button to play, click again to pause — only one clip plays at a time. Add a caption below the block in Notion to label it; the caption appears as a title next to the button (e.g. "Voiceline — 'I remember the night…'"). Video /video Paste a YouTube link and it will embed as a player on the wiki. Also works with direct video file URLs.

Bookmark /bookmark A styled link block. Good for artist portfolio links, social media pages, or external references. Add a caption to label it.

Text Formatting Inside a Block

Select any text inside a paragraph or heading to see the formatting toolbar. You can:

  • Make text bold (Ctrl+B), italic (Ctrl+I), or underlined (Ctrl+U)
  • Add a link to any selected text (Ctrl+K)
  • Use inline code style for things like item names or game terms

Common Use Cases

Character Backstory

Use a Toggle Heading labelled "Backstory" so readers can choose to expand it. Keeps the page clean for people who want to avoid spoilers.

Notable Session Events

Create a Toggle Heading for each session (e.g. "Session 12 — The Bridge Encounter"). Write what happened inside. Readers can browse without spoiling future sessions.

Voicelines

Use an Audio block. On the wiki it renders as a small circular play button — click to play, click again to pause. Only one clip plays at a time, so multiple voicelines on the same page won't overlap. The audio file must be publicly hosted (not just saved on your computer).

Labelling a voiceline: After inserting the audio block, click it in Notion and type in the caption field that appears below it. The caption becomes the label shown next to the play button on the wiki. Without a caption the button appears alone with no title, so it's good practice to always add one.

  • Upload to a file hosting service and paste the direct .mp3 or .wav link
  • If the voiceline is a YouTube clip, use a Video or Embed block instead

Artwork & Artist Credits

Use an Image block and fill in the caption with the artist's name and a link to their portfolio. If you want multiple pieces side by side, use a Columns layout with an Image in each column.

For linking directly to an artist's page, use a Bookmark block — it creates a clearly labelled link card.

Campaign Description / Overview

Create a special entry in your campaign's database with Category set to something like "Overview". Use the page body for a full description, session zero notes, tone, themes, etc.

Relationship Maps / Connection Lists

Use a Table block or a Bulleted List with bold names and descriptions to list a character's relationships.

Keeping Long Pages Readable

For entries with a lot of content, use Toggle Headings for each major section. Readers can collapse sections they've already read or don't want spoiled.

Adding Campaigns & Categories

You don't need to do any setup before creating a new campaign or category — just type the name when creating an entry.

New Campaign

  1. Create or open any entry in the database.
  2. Click the Campaign field.
  3. Type the new campaign name and select Create [name].

The new campaign will immediately appear in the wiki's campaign list.

New Category

  1. Same process — click the Category field on any entry and type a new name.
  2. The new category appears under its campaign automatically once at least one entry has it set.
Tip: Keep category names consistent — NPC and NPCs will show up as two separate categories on the wiki. Decide on a naming convention (plural is usually clearest) and stick to it.

Images & Media Tips

Profile Image (property)

The Profile Image property expects a direct image URL — a link that ends in .jpg, .png, .webp, or similar, and starts with https://.

Good sources for permanent image links:

  • Imgur — right-click an image → Copy image address
  • Discord — upload the image to a channel, right-click → Copy link (note: Discord CDN links may expire)
  • Any direct image host — the URL must point directly to the image file, not a webpage containing it
Tip: If you paste the URL into a browser address bar and you see only the image (nothing else on the page), it's a valid direct image URL.

Images in the Page Body

For images inside the page body (not the profile picture), you can either:

  • Upload directly to Notion — easiest, but Notion-hosted image URLs expire after ~1 hour. Fine for viewing, but the wiki may show broken images after a while if you use direct Notion uploads.
  • Use "Embed link" when adding the image block — paste an external URL. This won't expire.

For artwork that needs to stay reliably visible, host it externally (Imgur, etc.) and paste the link.
