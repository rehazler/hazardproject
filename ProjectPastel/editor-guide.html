How It Works

The Voidverse wiki is fully self-contained — all content is created, edited, and published directly on the site. There is no external tool or third-party CMS to log into. Everything lives in a Supabase database that the site reads and writes to directly.

The wiki is organised into three levels:

  • Campaigns — the top-level grouping (e.g. a campaign name). Each campaign has its own banner, icon, description, and accent colour.
  • Categories — sections within a campaign (e.g. NPCs, Locations, Items). Used to group related entries.
  • Entries — individual wiki pages. Each entry has a settings panel (name, profile image, banner, layout options) and a full content editor.
Tip: Changes are saved only when you click ✓ Save or press Ctrl+S. The editor auto-saves a local draft every 2 seconds while you type, but the draft is only a backup — always save to publish your changes.

The Editor Toolbar

When you are logged in as an editor, a toolbar appears at the top of the wiki page. It is only visible to you — visitors see the normal wiki without any editor controls.

ButtonWhat it does
✏ Edit Opens the content editor for the currently viewed entry. Only visible when you are on an entry page.
✓ Save Saves the entry content and settings to the database and publishes the changes. Appears only while editing.
✗ Cancel Discards any unsaved changes and exits edit mode. You will be prompted to confirm if there are unsaved changes.
📋 Templates Opens the template manager. Save the current editor content as a reusable template, or apply a saved template to the current entry.
📁 Media Opens the Media Library to upload files or browse and select previously uploaded assets.
⏻ Log out Signs you out and reloads the page.

A status message appears to the left of the buttons while you are editing — it shows the current block and word count, and any save or error messages.

Session: Your login persists across browser sessions — you normally only need to log in once per device. To sign out, click ⏻ Log out. You will be redirected to the login page to get back in.

Campaigns

Creating a Campaign

1
From the campaign list, click the + New Campaign button at the top.
2
Enter the campaign name in the prompt and click Create. The campaign appears in the list immediately.

Reordering Campaigns

Drag any campaign card by its body to reorder it in the list. The new order is saved automatically when you drop it.

Campaign Settings

Each campaign card has a ⚙ button. Click it to open the campaign settings panel.

FieldWhat it does
Name The displayed campaign name. Changing this also updates the URL slug.
Icon An emoji (e.g. 🐉) or a direct image URL shown alongside the campaign name in the list. Optional.
Description A short blurb shown on the campaign card. Optional.
Accent colour A highlight colour used for the campaign's card and themed elements.
Banner image URL A wide header image shown at the top of the campaign page. Can be typed in or chosen from the Media Library.
Banner display All the banner customisation controls — fit, height, zoom, opacity, and more. See the Banner Display section for full details.

Deleting a Campaign

A ✕ delete button is available on each campaign card. You will be asked to confirm by typing the campaign name before it is permanently removed. Deleting a campaign also removes all its categories and entries.

Warning: Campaign deletion is permanent and cannot be undone. All entries inside the campaign are deleted as well.

Categories

Categories group entries within a campaign (e.g. NPCs, Locations, Factions). They appear as collapsible sections inside the campaign view.

Creating a Category

Inside a campaign, click the + New Category button. Enter a name and confirm. The category appears immediately with no entries yet.

Category Settings

Each category pill has a ⚙ settings button. The settings panel lets you change the category Name and set an optional Icon — either an emoji (e.g. 🎲) or a direct image URL. The icon appears next to the category name in the wiki.

Deleting

A ✕ delete button is available on each category pill. You will be asked to type the category name to confirm before it is permanently removed. Deleting a category removes all entries inside it.

Reordering Categories

Drag any category pill left or right to reorder the categories within a campaign. The new order is saved automatically.

Tip: Keep category names consistent — NPC and NPCs are treated as two separate categories. Pick a convention (plurals are usually clearest) and stick to it.

Entries

An entry is a single wiki page — a character, location, item, or any piece of lore. Each entry has a settings panel (structured metadata) and a content area (free-form blocks).

Creating an Entry

1
Navigate to the campaign and category where the new entry should live.
2
Click + New Entry. Enter the entry name and confirm.
3
The entry opens in edit mode automatically. Fill in the settings panel and write the content, then save.

Editing an Entry

Open the entry you want to edit, then click ✏ Edit in the toolbar. The settings panel appears above the content editor. Make your changes and click ✓ Save or press Ctrl+S.

Deleting an Entry

From the category list, each entry card has a ✕ delete button. You will be asked to confirm before the entry is permanently removed.

Moving an Entry

Each entry card has a ↕ move button. Click it to choose another category within the same campaign to move the entry to. The entry disappears from the current list and reappears in the chosen category.

Duplicating an Entry

Each entry card has a ⎘ duplicate button. Clicking it creates an exact copy of the entry (all blocks, settings, and banner) with " (copy)" appended to the name. The copy opens immediately so you can edit it.

Reordering Entries

Drag any entry card up or down within a category to reorder it. The new order is saved automatically.

Entry Settings

When editing an entry, the settings panel appears at the top of the page above the content editor. These fields control the entry's metadata and appearance.

FieldWhat it does
Name The entry's title as displayed on the wiki.
Subtitle An optional tagline or descriptor shown below the name (e.g. "Archmage of the Eastern Tower").
Profile image URL A portrait image shown in the sidebar. Paste a direct image URL or click the browse button to choose from the Media Library.
Banner image URL A wide header image shown above the entry title. Paste a URL or pick from the Media Library. See Banner Display for customisation options.
Accent colour A highlight colour used throughout the entry page (links, headings, decorative elements).
Page style Full page (default) — standard layout. Section — a tighter, more contained layout for shorter entries.
Title divider A decorative element beneath the entry title. Options: None, Line only, Moon (split), Moon (full).
Divider colour The colour of the title divider — Default, Lavender, Mint, or Pink.
Divider thickness How thick the divider line renders — Very Thin, Thin, Medium, or Thick.
Sidebar Controls where the profile sidebar is placed: Right (default), Left, Float right, Float left, or Hidden (full-width, no sidebar).
Tip: Any change in the settings panel is treated as an unsaved change — you still need to save for it to take effect on the live wiki.

Banner Display Settings

The Banner Display controls appear automatically when a banner image URL is filled in. They let you fine-tune how the image is presented without touching any code. The live preview in the panel updates as you adjust each setting.

Drag inside the preview to reposition the image (sets the focal point).

Fit

OptionWhat it does
Fill (cover)Scales the image to fill the banner completely. Edges are cropped. Best for most images.
Fit insideScales the image so the whole image is visible. Letterboxed if the aspect ratios differ.
StretchForces the image to fill the banner exactly, ignoring aspect ratio. Can distort the image.
Actual sizeDisplays the image at its natural pixel size. A Zoom slider and drag-to-pan appear for precise framing.

Zoom (Actual size only)

Appears when Actual size is selected. Ranges from 25% to 400%. Use this alongside drag-to-pan to zoom into a specific part of the image.

Height

Sets a fixed pixel height for the banner: Short (120 px), Medium (180 px), Tall (240 px), or X-Tall (320 px). Click the active button again to remove the height constraint. The selected height is shown as a badge in the preview corner.

Image opacity

Controls how transparent the banner image itself is (0–100%). Useful when you want to layer an overlay tint on top or create a subtle washed-out effect. Does not affect overlay or text layers.

Border radius

Rounds the corners of the banner: None, Slight, Rounded, or Pill.

Overlay tint

Adds a colour wash over the banner image. Click the colour swatch to pick a colour, then toggle it On/Off. The opacity slider controls how strong the tint is.

Effects

EffectWhat it does
TileRepeats the image as a tiled pattern across the banner. Combines with any Fit mode.
Bottom fadeAdds a gradient that fades the bottom edge of the banner into the page background.
ParallaxThe banner scrolls at a slower rate than the page, creating a depth effect.

Text overlay

Adds text directly on top of the banner image. Type into the text field to enable it, then configure:

  • Position — a 3×3 grid for placing the text (top-left, center, bottom-right, etc.)
  • Size — Small, Medium, or Large
  • Colour — a colour picker for the text

Leave the text field empty to disable the overlay entirely.

Alt text

A text description of the banner image for screen readers and accessibility tools. Optional but recommended for important images.

Content Blocks

The content editor is block-based — every piece of content is a distinct block. Click into the editor area and start typing, or press / to open the block menu and search for a type.

Blocks can be reordered by clicking the drag handle (⠿) that appears to the left of each block and dragging it to a new position. Every block also has a Duplicate option in its settings menu (the three-dot icon) — click it to insert an exact copy of the block immediately below.

Text & Structure

BlockHow to insertWhat it does
Text Just start typing Regular body text. Supports bold, italic, inline code, links, and other inline formatting.
Heading Type #, ##, or ### then space — or use /heading Section headers at levels H2, H3, and H4. The block settings (⋮ or right-click the block) let you change the level, pick a colour (Default, Lavender, Mint, Pink, White), set alignment (Left, Centre, Right), toggle bold, and add a decorative divider (Full-width line, Text-width line).
Bulleted list Type - then space, or /list An unordered list. Press Tab to indent an item, Shift+Tab to outdent.
Numbered list Type 1. then space, or /numbered An ordered numbered list.
Quote Type " then space, or /quote A styled block quote with optional attribution text. Style options: None, Line (left border), Box, Shade. Separate alignment controls for the quote text and the attribution line (Left, Centre, Right each).
Callout /callout A highlighted box with a selectable variant (Info, Warning, Danger, Success) and an optional custom emoji. Good for DM notes, lore flags, or important warnings.
Toggle /toggle A collapsible section. Any other block type can be nested inside. The heading supports all the same options as the Heading block (level H2–H4, colour, alignment, bold, divider). Block settings also include an Outer border toggle and a Separator line between the heading and body. Great for backstory, spoilers, or long lore sections that readers can choose to expand.
Columns /columns A side-by-side multi-column layout. Starts with 2 columns; use the + Add column and − Remove last buttons to go up to 6. Each column can hold any other block type. The ⚙ icon on each column opens per-column settings: Width (relative fr units), Border (None, Lavender, Mint, Pink, Subtle), Background (None, Dark, Darker, Lavender/Mint/Pink tint), Padding (None, Small, Medium, Large), Radius, and Text align. A section label can be added above the whole block.
Table /table A data table. Toggle With headings in the block settings to treat the first row as a styled header row. Good for spell lists, item stats, timeline grids, or relationship charts.
Divider Type --- or /divider A decorative horizontal section separator. Block settings: Style (Line, Moon, Moon full), Thickness (Very Thin, Thin, Medium, Thick), and Color (Default, Lavender, Mint, Pink). A live preview updates as you adjust each option.
Spacer /spacer Adds blank vertical space between blocks. Adjust the height with the slider (10–400 px).

Media

BlockHow to insertWhat it does
Image /image Displays an image. Paste a URL, upload a new file, or choose from the Media Library. Supports a caption for context or credits.
Audio /audio A small themed play button that plays a hosted audio file. Paste a direct URL to an .mp3, .ogg, or .wav file, or upload one via the Media Library. Has two optional text fields: Caption (shown below the player as a description) and Section label (a small all-caps label displayed above the block, e.g. "SOUNDTRACK"). Only one clip plays at a time.

Link (Bookmark) /bookmark A styled link card. Paste the URL, add an optional display label (falls back to the URL if left blank), and an optional section label shown above the block. Good for artist portfolio links, social profiles, or external references.

Wiki-Specific Blocks

BlockHow to insertWhat it does
Properties /properties A structured key-value data block. Add rows with a label and value (e.g. Race / Human, Class / Wizard). Supports five display modes: Horizontal (compact stat boxes), Vertical (a list), Stat Attributes, Stat Block, and Class Wiki Table. An Align dropdown (Left, Centre, Right) controls the block's horizontal alignment. Use this for character sheets, item stats, or any tabular data. A section label can be added above the block.

Auto-save Draft

While you are editing, the editor automatically saves a local draft every 2 seconds after you stop typing. If you accidentally close the tab or navigate away, your unsaved work is preserved in the draft. The next time you open that entry in edit mode the draft will be restored.

Tip: A draft is a local backup only. Click ✓ Save to permanently publish your changes to the database.

Media Library

The Media Library stores uploaded images and audio files on Cloudinary, keeping all your media in one place. You can access it from the 📁 Media toolbar button, or from the browse button next to any image or audio URL field while editing.

Storage Bar

At the top of the Media Library a progress bar shows how much Cloudinary storage you have used. It turns yellow at 70% and red at 90%. The bar shows the exact amount used and the total limit (e.g. 45.2 MB / 25 GB used (0.2%)). The free Cloudinary tier includes 25 GB of storage and 25 GB of monthly bandwidth.

Uploading Files

1
Select a folder in the left panel (or stay in Uncategorized). Uploading while "All Files" is selected sends the file to Uncategorized.
2
Click ⬆ Upload and choose a file from your device.
3
The file uploads and appears in the library automatically. The active folder switches to the folder it was uploaded into.

Browsing & Selecting

Use the left panel to filter by folder. Each folder shows a file count and the total size of its contents. A search bar lets you filter by filename across all folders. Images show a thumbnail, audio files show a 🎵 icon. Each card also shows the file size when available.

When the picker is opened from a URL field (e.g. profile image or banner), clicking Use on a card inserts its URL into that field. Filter tabs (All / Images / Audio) narrow the picker to the type relevant to the field that opened it.

Copying a URL

Click 📋 Copy URL on any file card to copy the file's URL to the clipboard. The button briefly shows "✓ Copied" to confirm.

Moving Files Between Folders

Each file card has a folder dropdown at the bottom. Change the selection to move the file to a different folder. The URL never changes — only the library organisation is updated.

Deleting Files

Click ✕ Delete on a file card and confirm the prompt to permanently remove the file from Cloudinary. This cannot be undone. Any entries that reference the deleted file will show a broken image or missing audio.

Folders

Folders organise assets within the library but do not change URLs. All Files shows every asset regardless of folder. Files uploaded without a folder go into Uncategorized.

To create a new folder before uploading, click the + New Folder button at the bottom of the sidebar and enter a name. The folder becomes active immediately so your next upload goes into it. Folders with no files do not persist — they disappear when you reload the manager.

Tip: For images used as profile pictures or banners, consider using folders named after the campaign to keep things organised (e.g. "Campaign-Name/Characters").

Templates

Templates let you save the content and structure of an entry and reuse it across future entries. Click the 📋 Templates toolbar button to open the template panel at any time. The panel can be opened even when you are not editing — to browse, preview, or delete templates.

Saving a Template

The save form is only visible while you are in edit mode. If you open the Templates panel outside of edit mode, you will see only the list of saved templates.

1
While editing an entry, open the Templates panel.
2
Enter a name (required) and an optional description that reminds you what the template is for.
3
Choose a save mode:
  • Structure only — clears most text content but keeps the block structure, formatting, and heading text. Paragraph text, quote text, audio/image URLs, list items, and property values are cleared. Heading text, callout variant, and property row keys are preserved. Best for reusable page skeletons.
  • With content — saves the full entry exactly as it is, including all block text and settings.
In both modes the current entry settings (banner, accent colour, sidebar layout, etc.) are saved alongside the blocks.
4
Click 💾 Save current as template. The panel closes and the template appears in the list immediately.

Previewing a Template

Click the ▾ Preview button next to any template to expand a compact summary showing the number of blocks and a flow of their types (e.g. 5 blocks: Heading → Text → Properties → Divider → Text). Click ▴ Preview again to collapse it.

Applying a Template

The apply buttons — Overwrite, Append, and Smart — are only visible while you are in edit mode on an entry. If the Templates panel is opened outside of edit mode, only Preview and Delete are shown.

ButtonWhat it does
Overwrite Replaces the entire current editor content with the template's blocks. If the template has saved entry settings (banner, layout, etc.), those are also applied. You will be asked to confirm before anything is changed.
Append Appends the template's blocks to the end of the current content without removing anything. Entry settings are not affected. Useful for inserting a boilerplate section into an existing entry.
Smart Intelligently merges the template into the current content:
  • Properties block — if the current entry already has a Properties block, the template's property rows are merged into it. New keys are added; if the template has a value for an existing key, that value is updated.
  • Headings — a template heading is only inserted if no heading with the same text already exists, preventing duplicate section headers.
  • Everything else — appended to the end of the current content.
Entry settings are not affected by Smart mode.

After applying any mode, the editor shows the merged result. Nothing is saved to the database until you click ✓ Save.

Deleting a Template

Click the ✕ button on any template row and confirm the prompt. Template deletion is permanent and cannot be undone.

Images & Media Tips

Using the Media Library (Recommended)

The easiest way to add images is to upload them to the Media Library first. Click the browse button (📁) next to any image URL field, upload your file, and it will be inserted automatically. Uploaded files are hosted on Cloudinary and URLs never expire.

Using External URLs

You can also paste any direct image URL into a URL field. The URL must point directly to the image file (not a page that contains it) and should start with https://. If you paste the URL into a browser address bar and only the image appears with no surrounding webpage, it is a valid direct URL.

Reliable sources for external direct-link images:

  • Imgur — right-click an image → Copy image address
  • Cloudinary (via the Media Library) — always preferred
Avoid Discord CDN links — Discord attachment URLs expire and will show broken images after a short time.

Recommended Media Hosting by Type

TypeRecommendedNotes
Images Media Library (Cloudinary) Upload directly on-site. 25 GB free storage. URLs never expire.
Video YouTube (unlisted) No storage limit. Use the Embed block and paste the YouTube URL.
Audio / Voice clips Media Library or SoundCloud Upload short clips directly. For longer audio, SoundCloud embeds work well.

Profile Images

Profile images are displayed in the entry sidebar as portraits. Square or portrait-oriented images work best. Recommended minimum size: 300×300 px. The image is cropped to a square with rounded corners.

Banner Images

Banners are very wide and relatively short — a landscape image with an aspect ratio of roughly 3:1 to 6:1 works best. When using Fill (cover) fit, drag in the banner preview to set which part of the image stays centred when the banner is cropped.
